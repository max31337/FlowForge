# Contributing to FlowForge 🤝

While FlowForge is a personal project that I am actively developing on my own, **contributions are always welcome!** 🎉 If you feel inspired to improve or add to the project, feel free to contribute via issues or pull requests.

Please note that, as this is my personal project, I may choose to continue development in the direction I envision, even if contributions are made. However, your help is highly appreciated, and any contributions will make FlowForge better for everyone. Thank you! 🙏

Whether you want to contribute or not, your support is appreciated, and I’m thankful for anyone who decides to contribute to this project. 🌟

---

## How to Contribute 🛠️

We welcome all kinds of contributions, including but not limited to:

- 🐞 Bug reports and 🧩 feature requests
- 📝 Documentation improvements
- 💻 Code contributions (bug fixes, new features, etc.)
- 🔍 Testing and quality assurance
- 🎨 Design and UI/UX improvements

---

## Reporting Issues 📝

If you find a bug or have a feature request, please follow these steps:

1. **Check for Existing Issues**: Before opening a new issue, check the [issues list](https://github.com/max31337/flowforge/issues) to see if it has already been reported or addressed.
   
2. **Open a New Issue**: If the issue hasn’t been reported, open a new issue with the following template:
   - **Bug Report**: Include steps to reproduce, expected behavior, and actual behavior.
   - **Feature Request**: Describe the feature clearly with examples of how it would improve the application.

---
